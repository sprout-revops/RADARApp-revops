Tips for Best Results

Start with gold table

🐙 Publish Your Dashboard to GitHub Pages

Once Claude generates your HTML dashboard, follow these steps to publish it as a shareable web app — no server, no cost, accessible from any browser.

How it works: Claude generates a single .html file. You upload it to GitHub and enable GitHub Pages — it becomes a live URL you can share with your team. Anyone with the link can open it, enter their Databricks credentials, and see live data.
Step 1 — Generate your dashboard
1
Copy the Claude Prompt
Go to the Claude Prompt tab → click Copy Prompt. Open a new Claude conversation and paste it.
2
Ask Claude to build your dashboard
Tell Claude what you need, e.g.: "Build a monthly MRR and funnel dashboard by segment and pipeline." Claude will return a complete HTML file.
3
Save the HTML file
Copy Claude's output → paste into a text editor (VS Code or Notepad) → save as index.html.
Step 2 — Publish to GitHub Pages
4
Create a GitHub repository
Go to github.com → + → New repository. Give it a name (e.g. revops-dashboard). Set visibility to Private if your team has a paid GitHub plan, otherwise Public. Initialize with a README.
5
Upload your index.html
In the repo, click Add file → Upload files. Upload index.html. Commit the changes.
6
Enable GitHub Pages
Go to Settings → Pages. Under Source, select Deploy from a branch → main → / (root). Click Save.
7
Share your live URL
After ~1 minute your dashboard is live at:
https://[your-username].github.io/[repo-name]/
Share this link with your team. Each person enters their own Databricks credentials on first visit — saved to their browser only.
Step 3 — Update your dashboard
8
Push updates anytime
When you generate an improved version from Claude, replace index.html in the repo. GitHub Pages republishes automatically within ~1 minute.
git add index.html git commit -m "Update dashboard" git push origin main
⚠️ Security note: Never hard-code your Databricks PAT token inside the HTML file. The generated dashboard stores credentials in localStorage on the user's own browser — this means the PAT is never exposed in the source code. For team-wide access, each user connects with their own PAT.
💡 Tip — Auto-refresh: The generated dashboard includes a 🔄 Refresh Data button and an optional auto-refresh every 30 minutes toggle. Users can enable it to keep their dashboard live without manually reloading.
📤 How to Publish Your Dashboard

Built your dashboard with Claude? Follow these steps to publish it as a live shareable URL — for free, using GitHub Pages or Vercel — then submit it to RevOps to be added to the Dashboards gallery.

✅ Already built your dashboard? Good — you're in the right place. This tab covers how to get it live on the internet so anyone on the team can open it from a browser link, no setup required.

📌 Haven't built it yet? Go to 🛠️ DIY Dashboard first — pick your metrics, generate the Claude prompt, and download your project files. Then come back here.
Before You Start — What You Need
Your project files
The files Claude generated: fetch_data.py, dashboard.html, .env, .gitignore, refresh_and_push.bat, and the data/ folder.

pip install requests
Git

Before You Start

Do these first on your computer before touching GitHub.

A
Save Claude's files into a project folder
Create a new folder anywhere (e.g. my-team-dashboard) and save every file Claude generated into it. The structure should look like this:
my-team-dashboard/
  📄 fetch_data.py
  📄 dashboard.html
  📄 .gitignore
  📄 refresh_and_push.bat
  📁 .github/workflows/
    📄 refresh.yml
Create the .github\workflows\ subfolder manually before saving refresh.yml into it:
mkdir .github\workflows
If this is your first time using Git on this computer, set your identity once — open PowerShell anywhere and run:
git config --global user.email "you@sproutsocial.com" git config --global user.name "Your Name"
B
Add your Databricks credentials
Create a file called .env in the same folder. Paste this in and fill in your actual values:
DATABRICKS_HOST=adb-XXXXXXXXXXXXXXXX.XX.azuredatabricks.net DATABRICKS_HTTP_PATH=/sql/1.0/warehouses/XXXXXXXXXXXXXXXXX DATABRICKS_TOKEN=dapi...your-personal-access-token...
Find these in Databricks: SQL Warehouses → your warehouse → Connection Details. Generate a token under Settings → Developer → Access Tokens.
Never share this file — it's excluded from GitHub by .gitignore.
C
Pull your live data from Databricks
Open PowerShell inside your project folder, then run:
pip install requests python fetch_data.py
✅ You should see: Done! Saved [N] records to data/dashboard.json
If it errors, double-check your .env credentials.
Publish to GitHub + Vercel
1
Create an empty GitHub repository
Go to github.com → + → New repository. Give it a name (e.g. my-team-dashboard). Set to Private. Leave all other options as-is and click Create repository.
💡 Use a personal GitHub account — organization accounts require a paid plan for Vercel.
2
Push your files to GitHub
Right-click inside your project folder → Open in Terminal. Then run these commands — replace with your actual GitHub username and repo name:
git init git add . git commit -m "Initial dashboard" git branch -M main git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO-NAME.git git push -u origin main
💡 git add . stages everything at once. Your .gitignore automatically keeps .env out — it will never be uploaded.
⚠️ If push is rejected with "fetch first" error, run:
git pull --rebase origin main git push
3
Deploy to Vercel
Go to vercel.com → log in with GitHub → Add New → Project

Find your repo → click Import → leave all settings as-is → click Deploy

After ~1 minute your dashboard will be live at:
https://YOUR-REPO-NAME.vercel.app/dashboard.html
💡 Vercel auto-redeploys every time you push to GitHub — no manual steps needed.
4
Add GitHub Secrets for auto-refresh
The refresh.yml workflow runs on a schedule and pulls fresh data from Databricks automatically. It needs your credentials stored as GitHub Secrets:

Go to: Repo → Settings → Secrets and variables → Actions → New repository secret

Add these three — paste the same values from your .env:
DATABRICKS_HOST DATABRICKS_HTTP_PATH DATABRICKS_TOKEN
Once added, data refreshes automatically at 8:30 AM, 1:30 PM, and 5:15 PM daily. Vercel picks up each refresh automatically.
5
Confirm it's working
Trigger a manual refresh to test everything:

1. Repo → Actions tab → Refresh Dashboard Data → Run workflow
2. Wait ~2 minutes for the ✅ green checkmark
3. Open your Vercel URL with /dashboard.html
4. You should see a 🟢 Live badge and today's refresh timestamp
💡 Want data refreshed from your own machine too? Open refresh_and_push.bat, update the folder path at the top, then schedule it via Windows Task Scheduler to run at the same times. Both GitHub Actions and Task Scheduler back each other up.
